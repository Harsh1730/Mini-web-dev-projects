Abstraction is the concept of hiding the complex implementation details from the user and only providing the necessary details. It is one of the most important concepts of OOPs. Abstraction is the process of hiding the implementation details and showing only the functionality to the user. Abstraction is used to only show the necessary features of the object. Abstraction is achieved by using abstract classes and interfaces. An abstract class is a class that cannot be instantiated and is used to define the base class of other classes. 3. Inheritance

Inheritance is the mechanism in which one class acquires the properties and behavior of another class. It represents the Parent-Child relationship. It is one of the most important concepts of OOPs. Inheritance is used to define the relationship between two classes. Inheritance is achieved by using the extends keyword. The class that is inherited is known as the parent class or superclass. The class that inherits the other class is known as the child class or subclass. Inheritance is used for code reusability and method overriding.

a. Types of Inheritance:

1. Single Inheritance:

Single inheritance is a type of inheritance in which a class inherits only single class. It is one of the most important concepts of OOPs. 

//class A extends B

2. Multiple Inheritance:

Multiple inheritance is a feature in which a class can inherit from more than one superclass. However, Java does not support multiple inheritance with classes to avoid ambiguity (the Diamond Problem). Instead, multiple inheritance in Java is achieved using interfaces.

  • Java Does Not Support Multiple Inheritance with Classes:
    • *A class cannot inherit from multiple classes in Java using extends.
    • *This prevents conflicts when multiple parent classes have the same method signature.
  • Java Supports Multiple Inheritance Using Interfaces:
    • *A class can implement multiple interfaces using implements.
    • *Since interfaces only contain abstract methods (before Java 8) or default methods (after Java 8), no ambiguity arises.
  • Diamond Problem Solution:
    • *Since Java does not allow multiple class inheritance, it avoids the diamond problem, where a class could inherit the same method from two different parent classes.
//class A extends B

3. Hierarchical Inheritance:

Hierarchical inheritance is a type of inheritance in Java where multiple subclasses inherit from a single parent class. This means a single base class (superclass) acts as a common ancestor for multiple derived classes (subclasses). Multiple Child Classes: In hierarchical inheritance, multiple child classes share a common base class but have their own individual features.

  • *Code Reusability: The common properties and methods are inherited from the parent class, reducing code duplication.
  • *Method Overriding: Subclasses can override the methods of the parent class to provide specific implementations.
  • *Supports Polymorphism: Since multiple child classes inherit from the same parent, polymorphism can be achieved through method overriding.
  • *Java Does Not Support Multiple Inheritance: While a class can have multiple child classes, a subclass cannot inherit from multiple classes directly due to Javaâ€™s restriction on multiple inheritance (avoiding ambiguity).
  • *Super Keyword Usage: The super keyword is used to call parent class methods or constructors in subclasses.
//class A extends B

4. Hybrid Inheritance:

Hybrid inheritance is a combination of two or more types of inheritance, such as single, multilevel, hierarchical, and multiple inheritance. Java does not support hybrid inheritance with classes due to the restriction on multiple inheritance (to avoid ambiguity). However, it can be achieved using interfaces.

  • *Combination of Different Inheritance Types: It combines single, multilevel, hierarchical, and multiple inheritance.
  • *Java Restriction on Multiple Class Inheritance: Java does not allow multiple inheritance with classes, preventing hybrid inheritance directly.
  • *Achieved Using Interfaces: Hybrid inheritance can be implemented by using interfaces instead of classes.
  • *Avoids Diamond Problem: Since Java allows multiple inheritance through interfaces, it prevents method ambiguity.

Method overloading is a concept of having two or more methods with the same name in a class with different parameters. Method overriding is a concept of having two methods with the same name and same parameters in two different classes.

Static Keyword

The static keyword in Java is used for memory management mainly. It can be used with variables, methods, blocks, and nested classes. The static keyword belongs to the class than an instance of the class.

a. Static Variables: Static variables are also known as class variables. They are shared among all the instances of the class. Static variables are initialized only once, at the start of the execution. A single copy of the static variable is created and shared among all the instances of the class. Static variables are created at the start of the program execution and destroyed automatically when the program ends. 
//static int count=0;
b. Static Methods: Static methods are also known as class methods. They can be called without creating an instance of the class. Static methods can access static variables and static data directly. They cannot access non-static data members or call non-static methods directly. Static methods are used to create utility functions that perform tasks not dependent on instance creation. 
//static void display() { ... }
c. Static Blocks: Static blocks are used for static initializations of a class. This code inside the static block is executed only once: the first time the class is loaded into memory. A static block is also known as a static initialization block. 
//static { ... }
d. Static Classes: A static class is a class that cannot be instantiated. It can contain only static members. Static classes are used to create utility classes that contain static methods and variables. 
//static class MyClass { ... }
